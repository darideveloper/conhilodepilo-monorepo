# Design: Markdown Preview Typography

## Architecture Overview
The SimpleMDE editor renders its preview inside a div with the class `.editor-preview` (or `.editor-preview-side` when using the side-by-side view). Because these classes are specific to the editor, we can safely apply styles to them without leaking styles into the broader Unfold layout.

## Implementation Details
We will add a CSS block to `static/css/style.css`. The styles will cover:
- **Headings**: `h1`, `h2`, `h3` with appropriate font sizes, weights, and top/bottom margins.
- **Lists**: `ul` and `ol` with `list-style-type` (disc/decimal), left padding, and `display: list-item`.
- **Emphasis**: `strong` and `em` tags.
- **Code**: `code` and `pre` blocks with monospaced fonts and subtle backgrounds.
- **Blockquotes**: Border-left and indentation.

Example CSS structure:
```css
.editor-preview h1 {
    font-size: 1.5rem;
    font-weight: 700;
    margin-bottom: 0.5rem;
}
.editor-preview ul {
    list-style-type: disc;
    padding-left: 1.5rem;
}
/* ... etc */
```

## Trade-offs
- **Manual Maintenance**: Since we are manually re-applying styles that Tailwind removed, any changes to the desired "look" of Markdown must be updated in our CSS rather than via Tailwind classes (since we don't have control over the HTML tags generated by SimpleMDE inside the preview).
- **Consistency**: We will aim for a "GitHub-like" neutral style that fits well within the Unfold admin's clean aesthetic.

## Verification Plan
1.  **Visual Audit**: Open an `Event` change form, type Markdown in the description, and toggle the Preview button.
2.  **Breadth Check**: Verify that `h1` through `h3`, `ul`, `ol`, `bold`, and `italic` all render correctly.
3.  **Isolation Check**: Ensure that adding these styles does not change the appearance of headings or lists in the sidebar or other parts of the admin UI.
